ZiplineLoader

class ZiplineLoader(source)

Gets code from an HTTP server, or optional local cache or embedded filesystem, and handles with a receiver (by default, loads it into a zipline instance).

Loader attempts to load code as quickly as possible with concurrent network downloads and code loading.

Concurrency

This class uses two different coroutine dispatchers:

  • The dispatcher is the only dispatcher permitted to call functions on the returned Zipline instance. Each instance is thread-confined and this dispatcher must implement that enforcement.

  • The cacheDispatcher is the only dispatcher permitted to call functions on ZiplineCache. The cache is also thread-confined, and the two threads may be different.

In applications where multiple Zipline instances are used, but the instances all share a cache, there should be N + 1 dispatchers: one for the cache plus another for each application.
